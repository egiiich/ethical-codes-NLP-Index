# Шкала статусов

## 1. Статус наличия темы: `topic_status`

Используется ровно одно значение из списка ниже.

### `explicit_section`
Есть отдельный раздел, прямо посвящённый выбранной теме,
или раздел с очевидно эквивалентным названием и содержанием.

### `implicit_scattered`
Отдельного раздела нет, но релевантные положения есть в нескольких частях кодекса.

### `brief_mention`
Тема упоминается кратко, но без развёрнутых правил поведения.

### `absent`
Релевантные положения по выбранной теме не обнаружены.

### `unclear`
Нельзя уверенно классифицировать документ по теме.

---

## 2. Качество доказательной базы: `evidence_quality`

### `high`
Есть прямые и понятные подтверждения.

### `medium`
Подтверждения есть, но они не идеально прямые.

### `low`
Доказательная база слабая.

---

## 3. Статус обработки: `review_status`

### `not_started`
Документ (или связка документ × тема) занесён в реестр, но анализ ещё не выполнялся.

### `in_progress`
Анализ находится в работе.

### `completed`
Анализ завершён и пригоден для использования.

### `needs_human_review`
Есть ограничения, мешающие доверять результату без проверки человеком.

---

## 4. Готовность к синтезу: `synthesis_ready`

### `yes`
Карточка завершена и может использоваться для синтеза по соответствующей теме.

### `no`
Карточка пока не должна использоваться для сильных выводов.
