# Шкала статусов

## 1. Статус присутствия принципа: `principle_status`

Используется для каждого `principle_code` в каждом кодексе.

Допустимые значения (такие же, как раньше для статуса раздела):

### `explicit_section`
Есть отдельный раздел, где принцип сформулирован явно и операционально.

### `implicit_scattered`
Принцип не выделен в отдельный раздел, но устойчиво подтверждается несколькими фрагментами в разных частях кодекса.

### `brief_mention`
Принцип упомянут кратко, без развёрнутых правил применения.

### `absent`
Подтверждений принципа в кодексе не обнаружено.

### `unclear`
Нельзя надёжно определить статус принципа (фрагментарный текст, низкое качество извлечения, неоднозначность формулировок).

## 2. Статус обработки: `review_status`

### `not_started`
Документ занесён в реестр, но анализ ещё не выполнялся.

### `in_progress`
Анализ документа в работе.

### `completed`
Анализ документа завершён.

### `needs_human_review`
Есть ограничения, требующие ручной проверки.
